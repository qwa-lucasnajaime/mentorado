# Projeto de mentoria de novos membros da equipe
### Como resolver os exercícios propostos?
O primeiro passo é criar um fork deste repositório na sua conta. Para tal, siga as instruções dadas pelo próprio GitHub

O próximo passo é implementar as classes "ocas" que você vai encontrar dentro dos pacotes.

Após implementar o que for necessário, efetuar o push para o seu repositório (Não é necessário abrir um Pull Request de volta para o repositório principal) e passar a URL do seu repositório para quem for avaliar os exercícios

### Direcionamentos da implementação
Não é permitido alterar a estrutura inicial "forkeada" do repositório principal. Isso significa que não é permitido alterar os nomes das classes, assinaturas de métodos, pacotes onde estão as classes, nem renomear pacotes, etc. Caso essa orientação não seja seguida, os testes unitários criados pelos mentores irão falhar e por consequência não conseguirão avaliar a assertividade da sua implementação, portanto, cuidado com a base de código.

Entretanto é possível a criação de novos métodos e novas classes a fim de auxiliar a implementação, isto é, se achar que o método a ser implementado irá se beneficiar de uma quebra ou de um método ou classe auxiliar, é possivel criar essas novas estruturas.

Caso tenha alguma dúvida, não hesite em perguntar a qualquer um, estamos disponíveis para orientações e discussões.
